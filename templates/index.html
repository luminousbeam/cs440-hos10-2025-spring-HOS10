How to Write Your Own Mini SQAP

  1. Choose a real-world software failure — e.g., Mars Orbiter loss, Therac-25, Equifax breach, Boeing 737 MAX issue.
  2. Briefly describe the system — What was the software supposed to do?
  3. Define 1–3 quality objectives — What quality goals could have prevented the failure?
  4. Identify responsible roles — Who would ensure these objectives are met?
  5. List standards or practices — e.g., input validation, code review, penetration testing.
  6. Outline a basic test plan — What tests could catch the bug?
